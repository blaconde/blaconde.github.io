Data Quality Controls

Examples of data validation logic implemented in Python and SQL to detect inconsistencies, incorrect formats, and data quality issues.

Python SQL Data Quality Validation Logic

Project Overview

This section presents example validation rules used to detect data quality issues. The focus is on identifying incorrect formats, missing values, and inconsistencies that may impact analytical accuracy and business processes.

The examples below illustrate a simplified version of logic typically used in real-world data quality monitoring.

Email Validation

Python

import re

def validate_email(email):
    pattern = r"^[\w\.-]+@[\w\.-]+\.\w+$"
    return isinstance(email, str) and re.match(pattern, email) is not None

This validation checks whether the email follows a basic structural pattern. It ensures the value contains a local part, a domain, and a valid format using a regular expression.

Phone Number Validation

Python

def validate_phone(phone):
    return phone.isdigit() and len(phone) in [9, 11]

This rule verifies whether a phone number contains only digits and matches expected lengths. It helps detect incorrectly formatted or incomplete values.

Missing Value Check

Python

def is_missing(value):
    return value is None or value == ""

This validation identifies missing values in datasets. Detecting null or empty fields is essential for ensuring data completeness and reliability.

Duplicate Detection (SQL)

SQL

SELECT email, COUNT(*)
FROM customers
GROUP BY email
HAVING COUNT(*) > 1;

This query identifies duplicate records based on email. Duplicate detection is critical for maintaining clean datasets and avoiding incorrect analysis results.
